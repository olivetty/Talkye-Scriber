# Talkye Scriber — Release Process

When the user asks to create a new version/release, follow these steps exactly.

## Prerequisites
- Branch: `scriber` (never release from `main`)
- Remote: `origin` → `github.com/olivetty/Talkye-Meet-Assistant`
- AppImage build script: `build-appimage.sh` (project root)
- Version file: `app/lib/version.dart`

## Step-by-step Release

### 1. Bump version
Edit `app/lib/version.dart` — change `appVersion` to the new version (e.g. `'0.4.0'`).

### 2. Build AppImage
```bash
# From project root
./build-appimage.sh
```
This does everything: Flutter build, bundles Python/sidecar/whisper/sox, creates AppImage.
Output: `TalkyeScriber-x86_64.AppImage` in project root.

### 3. Test locally
```bash
# Kill old processes
pkill -f "talkye_app" 2>/dev/null
pkill -f "uvicorn.*server:app.*8179" 2>/dev/null

# Copy to test location
cp TalkyeScriber-x86_64.AppImage ~/Code/test/

# User tests manually
```

### 4. Commit and tag
```bash
git add -A
git commit -m "Release v0.X.0 — <short description>"
git tag v0.X.0
git push origin scriber
git push origin v0.X.0
```

### 5. Create GitHub Release
Use GitHub CLI (`gh`) or API:
```bash
gh release create v0.X.0 \
  --repo olivetty/Talkye-Meet-Assistant \
  --title "Talkye Scriber v0.X.0" \
  --notes "<release notes>" \
  TalkyeScriber-x86_64.AppImage
```

If `gh` is not installed:
```bash
# Install GitHub CLI
sudo apt install gh
gh auth login
```

Alternative: create release via GitHub API with curl (see below).

### 6. Verify
- Check https://github.com/olivetty/Talkye-Meet-Assistant/releases
- Ensure AppImage asset is attached
- Run old version of app — should show "Update vX.X.X" in status bar

## GitHub API fallback (if gh CLI unavailable)
```bash
# Create release
curl -X POST \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  https://api.github.com/repos/olivetty/Talkye-Meet-Assistant/releases \
  -d '{"tag_name":"v0.X.0","name":"Talkye Scriber v0.X.0","body":"Release notes here","draft":false}'

# Upload asset (get upload_url from response above)
curl -X POST \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Content-Type: application/octet-stream" \
  "https://uploads.github.com/repos/olivetty/Talkye-Meet-Assistant/releases/RELEASE_ID/assets?name=TalkyeScriber-x86_64.AppImage" \
  --data-binary @TalkyeScriber-x86_64.AppImage
```

## How auto-update works
- `app/lib/updater.dart` checks `https://api.github.com/repos/olivetty/Talkye-Meet-Assistant/releases/latest`
- Compares `tag_name` (e.g. `v0.4.0`) with `appVersion` in `version.dart`
- If newer: shows green "Update v0.4.0" button in status bar
- User clicks → downloads new AppImage → replaces current `$APPIMAGE` → restarts
- Only works when running as AppImage (dev mode shows nothing)

## File locations
- Version: `app/lib/version.dart`
- Updater logic: `app/lib/updater.dart`
- Update UI: `app/lib/status_bar.dart` (bottom bar, shows update button + progress)
- Build script: `build-appimage.sh`
- AppRun template: inside `build-appimage.sh` (Step 6)

## Changing app icon / tray icon
- App icon (window + desktop entry): `app-icon.png` (project root) or `app/assets/talkye-meet.png`
- Tray icons (system panel): `app/assets/tray-dark.png` and `app/assets/tray-light.png`
- GTK window icon: `app/linux/runner/my_application.cc` loads from binary dir → bundle → AppDir
- Desktop entry icon: auto-installed by `app/lib/desktop_integration.dart`
- To change: replace the PNG files, rebuild AppImage

## Sound themes
- `subtle` = synthetic beeps (generated by sox at runtime)
- `alex` / `emma` = pre-recorded voice files in `sidecar/sounds/<theme>/start.wav, stop.wav`
- `silent` = no sound
- Sound files bundled in AppImage at `usr/sidecar/sounds/`
